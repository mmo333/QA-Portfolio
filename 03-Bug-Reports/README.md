# Bug Reports

## 📌 Overview
This section contains structured bug reports identified during ERP and API testing activities.

---

## 🧩 Testing Context

Bugs were discovered during:
- ERP system testing (Odoo modules)
- API testing using Postman
- Feature validation based on requirements

---

## 🐞 Bug Reporting Format

Each bug is documented with:
- Clear title
- Steps to reproduce
- Expected vs actual result
- Severity & priority
- Business impact

### Bug 1: Missing validation in Purchase Order
- System allows creating purchase order without required supplier field

### Bug 2: API accepts invalid OTP format
- System does not validate OTP input correctly

### Bug 3: Inventory mismatch after transfer
- Stock quantity not updated after approval process
